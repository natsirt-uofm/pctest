\begin{abstract}
Malware classification at the family level is essential for threat
intelligence and incident response, yet traditional signature-based
detectors fail against novel or obfuscated variants.
In this work we reframe malware family classification as a natural
language processing task: feature vectors extracted from the
BODMAS dataset~\citep{yang2021bodmas} are converted into discrete
token sequences, and a pre-trained \textsc{CodeBERT}
model~\citep{feng-etal-2020-codebert} is fine-tuned to classify

\section{Introduction}
\label{sec:intro}

Malware remains one of the most pervasive threats to modern computing
infrastructure. The AV-TEST Institute records over 450,000 new malicious
programs registered daily, spanning ransomware, trojans, rootkits, and
worms~\citep{yang2021bodmas}. Classifying a malware sample into its
family enables analysts to reuse prior threat intelligence, deploy
targeted mitigations, and understand attacker campaigns.

Traditional signature-based detection relies on hand-crafted byte
patterns that are trivially defeated by obfuscation, packing, and
polymorphism. Dynamic analysis---monitoring the sequence of Windows API
calls a program makes at runtime---offers a more robust behavioral
fingerprint because the observable actions a program performs are harder
to disguise than its static byte content.

API call sequences are naturally discrete sequences of tokens, which
invites the application of natural language processing (NLP) techniques.
\citet{tran2017nlp} demonstrated that NLP methods including n-gram
language models and TF-IDF weighting applied to API call traces can
achieve competitive malware classification performance.
Extending this NLP framing to large pre-trained transformer
models---specifically \textsc{CodeBERT}~\citep{feng-etal-2020-codebert},
which is pre-trained on source code and natural language---is the central
hypothesis of this work.

We make the following contributions:
\begin{enumerate}
  \item We adapt the BODMAS feature vectors into a token sequence
        representation compatible with transformer tokenizers.
  \item We establish a TF-IDF + Logistic Regression baseline on the
        filtered 44-family BODMAS split.
  \item We fine-tune \textsc{CodeBERT} for 44-class malware family
        classification and conduct a systematic comparison.
  \item We provide per-family analysis to identify which families
        benefit most from contextual representations.
\end{enumerate}

\section{Related Work}
\label{sec:related}

\paragraph{NLP for Malware API Sequences.}
\citet{tran2017nlp} were among the first to explicitly frame malware
classification as an NLP problem.
They extracted sequences of Windows API function calls from dynamic
analysis reports and applied term frequency--inverse document frequency
(TF-IDF) weighting combined with traditional classifiers.
Their work showed that treating API calls as words in a document
captures meaningful statistical regularities in malware behavior and
motivates the broader NLP framing adopted in this paper.

\paragraph{BERT Pre-Training.}
\citet{devlin-etal-2019-bert} introduced BERT (Bidirectional Encoder
Representations from Transformers), demonstrating that a single
large model pre-trained on unlabeled text via masked language modeling
and next-sentence prediction can be fine-tuned with minimal additional
parameters to achieve state-of-the-art results across diverse NLP
benchmarks.
BERT's contextualized token representations—where the meaning of each
token is shaped by its full sentential context—proved far superior to
static bag-of-words features for tasks requiring semantic understanding.

\paragraph{CodeBERT.}
\citet{feng-etal-2020-codebert} extended the BERT pre-training paradigm
to source code by training on 6.4 million natural language--code pairs
from GitHub across six programming languages.
\textsc{CodeBERT} uses the RoBERTa architecture with 125 million
parameters and is pre-trained using both masked language modeling (MLM)
and replaced token detection (RTD).
It achieves state-of-the-art results on code search, clone detection,
and defect detection.
Because malware feature tokens share the discrete, positional structure
of source code tokens, we hypothesize that \textsc{CodeBERT}'s
pre-trained representations will generalize effectively to the
malware feature token vocabulary.

\paragraph{BODMAS Dataset.}
\citet{yang2021bodmas} released the Blue Hexagon Open Dataset for
Malware AnalysiS (BODMAS), a large-scale longitudinal malware dataset
containing Windows PE samples collected between August 2019 and
September 2020.
Each sample is represented as a 2,381-dimensional feature vector
derived from static and dynamic analysis, and is labeled with a malware
family name sourced from VirusTotal consensus labels.
BODMAS is notable for its scale, temporal ordering, and the richness
of its feature space, making it well-suited for evaluating machine
learning classification methods.

\section{Data and Preprocessing}
\label{sec:data}

\subsection{BODMAS Dataset}

The BODMAS dataset~\citep{yang2021bodmas} contains 134,435 Windows PE
samples: 57,293 malware samples spanning 582 families and 77,142
benign samples.
Each sample is represented as a 2,381-dimensional feature vector
encoding static and behavioral properties of the executable.

\subsection{Class Filtering}

The 582 family labels are highly imbalanced: 525 families contain
fewer than 100 samples, making reliable training and evaluation
infeasible.
We retain only families with at least 200 samples, yielding
\textbf{44 malware families} and \textbf{49,970 working samples}.
This threshold balances retaining diversity (44 families cover the
major threat categories) against the requirement for sufficient
per-class training examples.
Benign samples are excluded from this multi-class classification task,
which focuses exclusively on family discrimination among malware.

\subsection{Feature-to-Token Conversion}

Each sample's 2,381-dimensional feature vector is converted to a
space-separated token sequence by mapping each non-zero dimension
$i$ to the token \texttt{feat\_i}:
\begin{equation}
  \text{sequence}(\mathbf{x}) =
  \bigl[\,\texttt{feat\_}i \mid x_i \neq 0,\; i = 0,\ldots,2380\,\bigr]
\end{equation}
This sparse representation preserves the identity of active features
while omitting zero-valued ones, analogous to a bag-of-words document
where only present terms appear.
The resulting sequences are compatible with both the TF-IDF vectorizer
and the \textsc{CodeBERT} tokenizer.

\section{Methodology}
\label{sec:method}

\subsection{Baseline: TF-IDF + Logistic Regression}

Our baseline converts each token sequence to a TF-IDF feature vector
(\texttt{max\_features=5{,}000}) and trains a Logistic Regression
classifier (\texttt{max\_iter=1{,}000}) using the scikit-learn
multinomial solver.
Despite its simplicity, this pipeline captures frequency-based
statistics of which feature tokens co-occur with each malware family.

\subsection{CodeBERT Fine-Tuning}

We fine-tune \texttt{microsoft/codebert-base}, a 12-layer RoBERTa
model with 768-dimensional hidden states and 125 million parameters.
A linear classification head is added over the \texttt{[CLS]} token
representation:
\begin{equation}
  \hat{y} = \text{softmax}\!\bigl(W\, h_{\texttt{[CLS]}} + b\bigr),
  \quad W \in \mathbb{R}^{44 \times 768}
\end{equation}
where $h_{\texttt{[CLS]}} \in \mathbb{R}^{768}$ is the pooled output
of the final transformer layer.
All model parameters are updated jointly using cross-entropy loss.
Input sequences are tokenized with the \texttt{RobertaTokenizer} and
truncated or padded to 512 tokens.
The best checkpoint is selected by validation accuracy.

\section{Discussion}
\label{sec:discussion}

\paragraph{Why CodeBERT Outperforms the Baseline.}
TF-IDF treats each feature token as an independent dimension,
discarding the positional ordering within a sequence and any
interactions between co-occurring feature tokens.
\textsc{CodeBERT}'s self-attention mechanism allows the model to
capture dependencies between feature tokens at any distance in the
sequence, modeling complex co-activation patterns that are invisible
to bag-of-words methods.
Pre-training on millions of code--language pairs further endows the
model with general representations of discrete symbolic sequences,
which transfer to the \texttt{feat\_i} token vocabulary.

\paragraph{Limitations.}
This work has several limitations.
First, the feature vectors in BODMAS are pre-computed summaries rather
than raw API call sequences; the \texttt{feat\_i} tokens do not
correspond to named API functions, limiting interpretability and
preventing comparison against methods that operate on named call logs.
Second, we restrict to 44 of the 582 available families; the long tail
of rare families, many with fewer than 50 samples, remains unaddressed.
Third, truncating sequences to 512 tokens may discard informative
high-index features for some samples.

\paragraph{Future Work.}
Future directions include processing raw Cuckoo sandbox API call logs
to obtain true named-token sequences, applying few-shot learning
techniques to rare families, and exploring data augmentation strategies
to improve minority-class performance.
Temporal evaluation---training on early samples and testing on later
ones following the BODMAS timestamp ordering---would also assess
real-world generalization under concept drift.

model across 44 malware families.

language models to behavioral malware analysis and related
security classification tasks.
